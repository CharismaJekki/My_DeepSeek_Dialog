# My DeepSeek 需求规格说明书

**版本：** V1.0
**日期：** 2025年3月
**项目名称：** My DeepSeek - AI 智能对话平台
**项目状态：** 已完成

---

## 1. 项目概述

### 1.1 项目背景

随着 AI 技术的发展，智能对话应用已成为人机交互的主流形式。本项目旨在构建一个仿 DeepSeek 风格的 AI 智能对话平台，为用户提供流畅、智能的对话体验。

### 1.2 项目目标

- 构建一个功能完整的 AI 对话应用
- 支持多模型切换（DeepSeek V3 / R1）
- 实现用户认证与对话历史管理
- 提供流式输出的对话体验

### 1.3 项目范围

| 范围       | 说明                          |
| ---------- | ----------------------------- |
| 核心功能   | AI 对话、多模型切换、对话管理 |
| 用户系统   | 用户注册/登录（Clerk）        |
| 数据持久化 | PostgreSQL + Drizzle ORM      |
| 平台       | Web 应用（响应式设计）        |

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 用户认证

| 功能          | 描述                 | 优先级 |
| ------------- | -------------------- | ------ |
| 用户注册/登录 | 支持用户登录         | P0     |
| 会话管理      | 自动管理用户登录状态 | P0     |

#### 2.1.2 用户权限

- 未登录用户可查看首页，但无法发起对话
- 登录后可创建对话、查看历史记录
- 用户只能访问自己的对话数据

---

### 2.2 对话模块

#### 2.2.1 创建对话

| 功能     | 描述                       | 优先级 |
| -------- | -------------------------- | ------ |
| 新建对话 | 用户输入问题后自动创建对话 | P0     |
| 对话标题 | 自动将首个问题作为对话标题 | P0     |
| 模型选择 | 支持 DeepSeek V3 / R1 切换 | P0     |

#### 2.2.2 对话交互

| 功能     | 描述                   | 优先级 |
| -------- | ---------------------- | ------ |
| 发送消息 | 用户输入问题并发送     | P0     |
| AI 回复  | 流式输出 AI 响应       | P0     |
| 模型切换 | 对话过程中切换 AI 模型 | P1     |

#### 2.2.3 流式输出

- 使用 Server-Sent Events (SSE) 实现实时流式输出
- 打字机效果展示 AI 回复
- 支持中断连接处理

#### 2.3.1 对话列表

## 3. 非功能需求

### 3.1 性能需求

| 指标          | 要求                    |
| ------------- | ----------------------- |
| 页面加载时间  | < 2 秒                  |
| AI 首字节响应 | < 500ms                 |
| API 响应时间  | < 200ms（不含 AI 调用） |

### 3.2 可用性需求

- 响应式设计，支持 PC 端访问
- 简洁直观的用户界面
- 清晰的错误提示

### 3.3 安全需求

| 需求     | 说明                         |
| -------- | ---------------------------- |
| 身份认证 | 所有 API 接口验证用户身份    |
| 数据隔离 | 用户只能访问自己的对话       |
| 敏感信息 | API 密钥存储在服务端环境变量 |

## 4. 数据模型设计

### 4.1 数据库表结构

#### 4.1.1 Chats 表（对话表）

| 字段名  | 类型   | 说明       | 约束       |
| ------- | ------ | ---------- | ---------- |
| id      | SERIAL | 对话 ID    | 主键、自增 |
| user_id | TEXT   | 用户 ID    | 非空       |
| title   | TEXT   | 对话标题   | 非空       |
| model   | TEXT   | 使用的模型 | 非空       |

#### 4.1.2 Messages 表（消息表）

| 字段名  | 类型    | 说明     | 约束                   |
| ------- | ------- | -------- | ---------------------- |
| id      | SERIAL  | 消息 ID  | 主键、自增             |
| chat_id | INTEGER | 对话 ID  | 外键 → chats.id        |
| role    | TEXT    | 消息角色 | 非空（user/assistant） |
| content | TEXT    | 消息内容 | 非空                   |

### 4.2 ER 图

```
┌─────────────┐       ┌─────────────┐
│    chats    │       │   messages  │
├─────────────┤       ├─────────────┤
│ id (PK)     │       │ id (PK)     │
│ user_id     │       │ chat_id(FK) │
│ title       │──1:N──│ role        │
│ model       │       │ content     │
└─────────────┘       └─────────────┘
```

## 5. API 接口设计

### 5.1 接口列表

| 接口              | 方法 | 功能                   | 认证 |
| ----------------- | ---- | ---------------------- | ---- |
| /api/chat         | POST | 发送消息，获取 AI 回复 | ✅   |
| /api/create_chat  | POST | 创建新对话             | ✅   |
| /api/get_chat     | POST | 获取单个对话           | ✅   |
| /api/get_chats    | POST | 获取对话列表           | ✅   |
| /api/get_messages | POST | 获取消息历史           | ✅   |

### 5.2 接口详情

#### 5.2.1 POST /api/chat

发送消息并获取 AI 回复（流式）

**请求参数：**

```json
{
  "messages": [...],
  "model": "deepseek-v3",
  "chat_id": 1,
  "chat_user_id": "user_xxx"
}
```

**响应：** SSE 流式输出

---

#### 5.2.2 POST /api/create_chat

创建新对话

**请求参数：**

```json
{
  "title": "对话标题",
  "model": "deepseek-v3"
}
```

**响应：**

```json
{
  "id": 1
}
```

---

#### 5.2.3 POST /api/get_chats

获取当前用户的所有对话

**响应：**

```json
[
  { "id": 1, "title": "问题1", "model": "deepseek-v3" },
  { "id": 2, "title": "问题2", "model": "deepseek-r1" }
]
```

## 7. 业务流程

### 7.1 新建对话流程

```
用户输入问题 → 检查登录状态 → 未登录跳转登录页
        ↓
已登录 → 调用 /api/create_chat 创建对话
        ↓
跳转到 /chat/{id} 页面
        ↓
调用 /api/chat 发送消息
        ↓
流式接收 AI 响应并展示
```

### 7.2 消息交互流程

```
用户输入消息 → 点击发送
        ↓
调用 /api/chat 接口
        ↓
前端建立 SSE 连接
        ↓
流式接收 AI 响应
        ↓
前端逐字展示（打字机效果）
        ↓
消息存储到数据库
```
